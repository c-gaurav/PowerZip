Creating a new ZIP File

Go to top

User can create a new archive by selecting New Archive from the PowerZip File pull-down menu or by clicking the New toolbar button. This activates the Save dialog box.

First, the user has to type the name of the ZIP file he wants to create (not the names of the files he want to compress). This dialog box works similarly to standard Windows file dialog boxes, so user can choose the drive and folder where the archive will be created. The names of any existing archives on the selected drive and folder are listed to help user choose a name that is not already in use.

Normally after creating an archive the user will want to add files to the archive, so then automatically activated the Add dialog box. In the Add dialog box, select the drive from drive pull-down box, then the directory in the box below and double click it. Then select one or more file or folders and click the (>) button. This will add those files to the Files to add box that will be added to the archive when Add button is clicked. To remove files from the Files to add box, select them and click (<) button. To remove all, user can click the (<<) button.

 

>To view the description of file, there is an option to select it in the Files to add box and click File Desc... button. If the user wants to put all the files in the directory and subdirectories in the same arrangement to a ZIP archive, he should check Include Sub Folders checkbox.

If user need backup to the files that are to be added, he can check Create Backup checkbox and if he wants to remove the original files on compression, he can check Delete on Compression checkbox.

Note: One can select multiple files in both Select File box and Files to add box using normal Windows file selection conventions.

Opening a ZIP or JAR File

Go to top

To open an archive, the user can use the Open toolbar button or select Open Archive from the PowerZip File pull-down menu to activate the standard Open dialog box.

Then, select the ZIP file he wants to open in the Open dialog box. This dialog box works just like the Open dialog boxes in other applications: just select the file you want to open from the list of files.

Once an existing ZIP archive is open user can extract the files in the archive. The files in a ZIP archive are listed in a list box in the main PowerZip window.

  • All the file names, with details, stored in the archive can viewed using the scroll bar or arrow keys.

Extracting Files

Go to top

When one extracts a file, PowerZip decompresses it and places it in the folder of his choice. He can restore whole folders and retain the structure of any sub-folders.

If he selects Extract from the Actions pull-down menu or clicks the Extract toolbar button, PowerZip lets him choose the folder where he'd like to place the extracted files by opening an Extract to dialog box.

The Extract to dialog box provides the following options:

  • The Selected files option button instructs PowerZip to extract only files that are selected in the main PowerZip window. User can use conventional file selection methods to choose files. This option is disabled if no files are selected.

  • The All files option button instructs PowerZip to extract all files from the archive.

  • The Files option button and edit field allow user to specify which files are extracted using filenames.

  • The Overwrite files checkbox determines whether existing files are overwritten with files from the archive.

Adding Files to an existing ZIP archive

Go to top

User can add files to an existing archive he has opened or to a new archive he has created, as in the previous section. If the Add dialog box is not displayed, he can select Add from the Actions pull-down menu or click the Add toolbar button.

>In the Add dialog box, select the files one wants to add to the ZIP archive. One can select multiple files using normal Windows file selection conventions. Once this is done, click the Add button to add the files. Once PowerZip finishes this, it will update the file list in the main window to reflect the new files in the archive.

Removing Files from an existing ZIP archive

Go to top

PowerZip can remove selected files from within an existing archive, or delete the entire archive. Delete from the Actions menu deletes the selected. Select entries from the list box in an open archive, then choose the Delete from the Actions pull-down menu or click the Delete toolbar button.

Note: These operations cannot be undone.

Renaming an archive

Go to top

Renaming of archives can be done by clicking on the Rename Archive menu item in File menu. This invokes a new window which requires the path of the file to be renamed and the new name. And on clicking Rename Archive button the system renames the name of the given file with the desired name entered.

Finding a substring in the name field of an opened archive

Go to top

This is a very useful tool, which helps to locate a file entry in a large zip files. This can be used to locate a particular type of files if the extension is searched. This operation can be performed by clicking on Find menu item in Edit menu. Then appears a new window with a text field and a find button. The user needs to enter the text he wants to search. When he clicks on the find button the system searches for the occurrences of the substring in the name field of the window and sets all the corresponding rows as selected.

Encrypt or Decrypt a archive

Go to top

This feature can encrypt or decrypt a zip file to send it on the network or store it safely. This can be done by clicking on the Encrypt menu item in Action Menu. This invokes a window which presents option of encryption or decryption. On selecting whether to encrypt or decrypt a file dialog appears. Then the user needs to select the file to perform the operation on. Then a confirmation window appears, on clicking yes button the user allows the operation to take place.

Using open with

Go to top

This feature works only when an archive is open and only one table entry is selected. Then the user needs to select with which application he wants to open the file with. The user needs to have an idea of the file format; if he selects the correct application then the file is extracted temporarily and opened with the selected application. Else the system gives an error “Format not supported”. There are three types of application to select from.
